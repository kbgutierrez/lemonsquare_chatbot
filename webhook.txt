🗑️ Data Cleanup (Phase 1)
DELETE /api/chat/sessions/{session_id}

Scope: Soft-deletes a chat session for privacy or cleanup.

Frontend Action: Pass the session_id in the URL. On a successful response, immediately remove the session from the UI sidebar and clear the active chat view.

📊 Analytics & Sync (Phase 2)
GET /api/analytics/summary

Scope: Fetches high-level system KPIs. Returns a JSON object with counts for active chats, escalated chats, embedded PDFs, manual rules, AI-learned chats, and fully synced tickets.

Frontend Action: Call this on page load for the Admin Dashboard to populate the top-level metric widgets.

POST /api/tickets/bulk-sync

Scope: Safely syncs missed tickets from the Helpdesk database into the AI's brain in the background.

Frontend Action: Attach to a "Force Sync" button. Since it runs in the background, display a success toast notification immediately so the user knows the process started.

♻️ Knowledge Restoration (Phase 4)
POST /api/documents/manual/{entry_id}/restore

Scope: Un-archives a deleted manual rule and tells the AI to re-learn it (re-embeds to Qdrant).

Frontend Action: Attach to a "Restore" button in an Archive/Trash view. Pass the entry_id in the URL.

POST /api/self_knowledge/chats/{session_id}/restore

Scope: Un-archives a deleted AI-learned chat summary and tells the AI to re-learn it.

Frontend Action: Pass the session_id in the URL. Refresh the knowledge base UI list upon success.
